# 🤖 EduCore Agent Strategy

This project utilizes a multi-agent orchestration strategy inspired by `wshobson/agents` to automate development, security, and quality assurance.

## 👥 Specialized Agents

| Agent | Tier | Role | Focus Area |
| :--- | :--- | :--- | :--- |
| **Architect-Pro** | Tier 1 (Opus) | System Design | Supabase Schema, Next.js App Router, Middleware |
| **Frontend-Wizard** | Tier 2 (Sonnet) | UI/UX Development | Tailwind CSS, Framer Motion, Radix UI |
| **Supabase-Expert** | Tier 2 (Sonnet) | Data & Auth | RLS Policies, Edge Functions, SSR Auth |
| **Security-Auditor** | Tier 1 (Opus) | Risk Assessment | Env secrets, SSR vulnerabilities, Twilio/Cloudinary leaks |
| **QA-Automator** | Tier 3 (Haiku) | Testing | Zod validation, Form logic, Prop types |

## 🛠️ Custom Workflows (Virtual Plugins)

We follow the **Conductor** pattern for task execution:

1.  **Context**: Update `.agent-context` with current project state.
2.  **Spec**: Generate technical specifications for new features.
3.  **Implement**: Multi-agent execution of the plan.
4.  **Verify**: Run `npm run lint` and `npm run type-check`.

## 📜 Progressive Disclosure Skills
